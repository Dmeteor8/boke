# ProcessSection 组件

展示合作流程的区块组件，使用时间线样式呈现四个合作阶段。

## 功能特性

- ✅ 渲染四个合作阶段
- ✅ 使用时间线样式展示流程
- ✅ 响应式布局（移动端垂直，桌面端水平）
- ✅ 步骤编号和连接线视觉效果
- ✅ 滚动进入视口时的渐显动画
- ✅ 悬停效果增强交互体验

## 需求追溯

- **需求 4.1**: 按顺序展示四个合作阶段
- **需求 4.2**: 显示阶段一："需求沟通与方案评估"
- **需求 4.3**: 显示阶段二："出具功能说明与报价"
- **需求 4.4**: 显示阶段三："分阶段开发与测试"
- **需求 4.5**: 显示阶段四："验收确认后正式上线"
- **需求 4.6**: 使用时间线或步骤图形式呈现

## 使用方法

### 基础用法

```tsx
import { ProcessSection } from './components/ProcessSection';
import type { ProcessStep } from './types';

const steps: ProcessStep[] = [
  {
    id: 'step-1',
    order: 1,
    title: '需求沟通与方案评估',
    description: '深入了解您的业务需求，评估技术可行性，提供初步解决方案建议',
  },
  {
    id: 'step-2',
    order: 2,
    title: '出具功能说明与报价',
    description: '根据需求整理详细功能清单，明确开发范围，提供透明的项目报价',
  },
  {
    id: 'step-3',
    order: 3,
    title: '分阶段开发与测试',
    description: '按照约定的时间节点推进开发，定期同步进度，确保功能符合预期',
  },
  {
    id: 'step-4',
    order: 4,
    title: '验收确认后正式上线',
    description: '完成功能验收，提供上线指导和基础维护，确保系统稳定运行',
  },
];

function App() {
  return <ProcessSection steps={steps} />;
}
```

## Props

### ProcessSectionProps

| 属性 | 类型 | 必填 | 说明 |
|------|------|------|------|
| steps | ProcessStep[] | 是 | 合作流程步骤数组 |

### ProcessStep

| 属性 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | string | 是 | 步骤唯一标识符 |
| order | number | 是 | 步骤顺序编号（1-4） |
| title | string | 是 | 步骤标题 |
| description | string | 是 | 步骤描述 |

## 响应式设计

### 移动端（< 1024px）
- 垂直时间线布局
- 步骤编号圆圈在左侧
- 垂直连接线
- 卡片内容在右侧

### 桌面端（≥ 1024px）
- 水平时间线布局
- 步骤编号圆圈在顶部
- 水平连接线
- 四列网格布局

## 视觉设计

### 颜色方案
- **连接线**: 蓝色渐变（from-blue-200 via-blue-400 to-blue-600）
- **步骤编号**: 蓝色渐变背景（from-blue-500 to-blue-700）
- **卡片**: 白色背景，阴影效果
- **文字**: 标题深灰色（gray-900），描述中灰色（gray-600）

### 动画效果
- **进入动画**: 从下方淡入（opacity + y轴位移）
- **延迟**: 每个步骤延迟 0.15s，形成连续效果
- **悬停**: 阴影增强效果

### 间距和尺寸
- **步骤编号圆圈**: 64x64px
- **卡片内边距**: 移动端 24px，桌面端 32px
- **区块上下边距**: 64px
- **步骤间距**: 移动端 32px，桌面端 16px

## 注意事项

1. **步骤数量**: 组件设计为展示 4 个步骤，如需更多步骤，可能需要调整布局
2. **order 属性**: 确保 order 值从 1 开始连续递增
3. **描述长度**: 建议描述文字控制在 50 字以内，保持简洁
4. **动画性能**: 使用 Framer Motion 的 viewport 优化，仅在进入视口时触发动画
